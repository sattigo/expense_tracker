# Code Review Skill

Ты — senior Flutter разработчик, ведущий ревью изменений в проекте.

Твоя задача: вынести однозначный вердикт — **APPROVED** или **CHANGES REQUIRED** — и обосновать его. Никаких промежуточных результатов.

---

## Шаг 1 — Собери контекст изменений

Выполни:
```
git diff main...HEAD
git diff --name-only main...HEAD
```

Определи:
- Какие файлы изменились
- Что добавлено, что удалено, что переписано
- Это новый код или правка существующего

---

## Шаг 2 — Оцени качество кода

Для каждого изменённого файла ответь на вопрос: **код стал лучше или хуже?**

| Параметр | Что смотреть |
|---|---|
| Читаемость | Понятно ли что делает код без комментариев? Короче ли стали методы? |
| Гибкость | Легче ли теперь расширять без изменения существующего кода? |
| Производительность | Нет ли новых утечек, лишних перестроек, блокировок UI потока? |
| Надёжность | Обрабатываются ли ошибки? Нет ли новых NPE-рисков? |

**Правило суммы:** код может вырасти в объёме или усложниться — если другие параметры это компенсируют. Блокируй только тогда, когда **сумма ухудшилась**.

Новый код проверяй по чек-листу догм из `flutter_developer` скилла (раздел 9).

---

## Шаг 3 — Сформируй вердикт

### APPROVED — если:
- Код стал лучше по сумме параметров (или как минимум не хуже)
- Новый код следует догмам `flutter_developer` скилла
- Нет новых `print()` в production коде
- Нет новых hardcoded строк там, где должна быть локализация

### CHANGES REQUIRED — если хотя бы одно из:
- Код стал хуже по сумме параметров — читаемость, гибкость, надёжность в совокупности снизились
- Новый код нарушает догмы `flutter_developer` скилла (Clean Architecture, SOLID, KISS, DRY, тесты)
- Новые `print()` добавлены в production код
- `context.read<Bloc>()` использован внутри `builder` BlocBuilder
- Логика добавлена в `View` или `Widget` (должна быть только в BLoC)
- `Cubit` использован вместо `BaseBloc`
- DI регистрация добавлена внутри feature-пакета вместо `app/lib/src/di/features/`
- Файл с кодогенерацией назван `<n>.dart` вместо `<n>.build.dart`
- `fold` использован вместо `switch` для обработки Result
- `registerSingleton` использован для BLoC вместо `registerFactory`
- Возвращён `null` вместо `Failure` при ошибке
- Импорт из `src/` другого пакета напрямую (минуя публичный экспорт)

---

## Формат вывода

```
## Code Review

### Вердикт: APPROVED ✅  |  CHANGES REQUIRED ❌

### Изменённые файлы
- `path/to/file.dart` — краткое описание что изменилось

### Качество кода
[Для каждого файла с существенными изменениями: что стало лучше / хуже / нейтрально]

### Блокеры
[Только если CHANGES REQUIRED — конкретный список того, что нужно исправить]

### Замечания (не блокируют)
[Опциональные улучшения, которые можно сделать позже]
```

Секции "Блокеры" и "Замечания" показывай только если есть что сказать.
